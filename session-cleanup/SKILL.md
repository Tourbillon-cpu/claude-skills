# 会话收尾清理

## 触发条件
当对话进入收尾阶段时自动触发——用户说"好了"、"就这样"、"下次再说"、"辛苦"、"88"等，或者单纯闲聊几句时，在回应最后附加一句清理提醒。

不需要用户主动提，也不需要每次都必须执行清理——只是提醒一下，让用户自己决定。

## 执行流程

### Step 1：回顾本次会话创建的文件
按类别列出：

```
📁 保留（自动识别）
├── 知识库笔记 → KnowledgeStorage/**/*.md
├── 脚本文件   → *.py
├── 导出图片   → *.png / *.pdf
├── 记忆文件   → memory/*.md
└── 已安装的 skill

🗑️ 可清理
├── 缓存文件   → %TEMP% 下的 .html、下载的网页等
├── git clone  → /tmp/repo* 等临时仓库
├── 用完了的指引 → 设置指南、README 等
└── 中间产物   → 测试输出、失败的导出、.bak 备份
```

### Step 2：询问用户
简洁一句话，不啰嗦。例如：
```
会话收尾提醒 📋 本次产生了几个临时文件（缓存/git clone/中间脚本），需要清理吗？
```

或者如果没什么可删的：
```
本次会话没产生什么临时文件，桌面挺干净的 ✅
```

### Step 3：执行删除
用户确认后删除，逐条执行，防止某条失败导致全部中断。删除前检查文件是否存在。

## 清理规则

### 自动归为可删的
- `%TEMP%`、`/tmp` 下的文件
- `git clone` 的仓库
- 名字含 `bak`、`backup`、`temp`、`cache` 的文件
- 网页抓取缓存（如 `wechat*.html`）
- 用户明确说不要的文件
- **确认已用完的导出图片**（如已发送、已用过的图）

### 自动归为保留的
- `KnowledgeStorage/**/*.md`
- `G:\大学课业\**` 下的文件
- `.py` 脚本文件
- `memory/*.md`
- `.claude/skills/*/SKILL.md`

### 不确定的 → 问用户
- 分析笔记
- 生成图片
- 项目文件（.opju）

## 注意事项
- 只删本次会话产生的文件
- 删除前让用户确认，用户说"不用"就不删
- 删完告诉用户删了啥，让用户放心
